🔧 所有指令與用法對照表

按功能分類：行程生命週期／排程／記憶體／IPC／信號量／除錯工具

指令語法 / 參數說明與使用時機範例
lsls列出內建的程式名稱 (hello, count5, factorial, ipc_sender, ipc_receiver, mutex_demo, calc_add 等)$ ls
runrun <程式名> [優先權]建立行程並放入就緒佇列 (Ready)，預設優先權為1，可自訂數值(數字越大優先越高)。$ run count5 3
$ run factorial
startstart [量子數]開始執行排程器，可選時間量子(預設1)。行程將按排程演算法執行。$ start 2
$ start
psps顯示所有行程狀態 (Ready / Running / Waiting / Terminated) 與 PID。$ ps
toptop動態顯示 CPU 使用次數、上下文切換次數、等待時間等統計資訊。$ top
killkill <PID>終止指定的行程 (可終止 Ready, Waiting, Running 狀態的行程)。$ kill 2
resetreset重置整個系統，清除所有行程、釋放記憶體、重置信號量與 IPC 狀態。$ reset
schedsched <演算法>變更排程演算法：PRIORITY / SJF / FIFO。影響下次 start 後的執行順序。$ sched SJF
$ sched PRIORITY
agingaging on / off啟用/停用老化機制。啟用後低優先權行程會逐漸提升優先權，避免飢餓。$ aging on
memmem顯示記憶體使用狀況：已用頁框數、缺頁次數、分頁表摘要。$ mem
loadload <檔案.asm> [優先權]從 programs/ 目錄載入組合語言原始檔，編譯成內部指令並產生行程。$ load programs/count5.asm 2
ipc_senderrun ipc_sender [pri]IPC 範例 — 傳送端行程，發送整數 42 給接收端。$ run ipc_sender
ipc_receiverrun ipc_receiver [pri]IPC 範例 — 接收端行程，等待並印出接收到的數值。$ run ipc_receiver
sem createsem create <初始值> <名稱>建立一個命名信號量，初始值通常為1表示互斥鎖。$ sem create 1 mutex
semsem顯示所有信號量狀態 (名稱、值、等待佇列)。$ sem
mutex_demorun mutex_demo [pri]使用信號量實作互斥的範例程式，展示 WAIT/SIGNAL 阻塞與喚醒。$ run mutex_demo 2
lsasmlsasm列出 programs/ 目錄下所有 .asm 組合語言原始檔。$ lsasm
disasmdisasm <程式名>反組譯已載入的程式，顯示指令位址與機械碼對照。$ disasm factorial
historyhistory顯示最近執行的指令歷史記錄 (含編號)。$ history
!n!<數字>重新執行歷史記錄中第 n 條指令。$ !2
deadlockdeadlock [resolve]偵測目前系統是否有死結 (循環等待)。加上 resolve 嘗試解除。$ deadlock
$ deadlock resolve
tuitui全螢幕儀表板，即時顯示行程、記憶體、IPC、信號量，按 q 退出。$ tui
calc_addrun calc_add <數字>內建示範程式：計算加法回傳 (實際效果視指令集而定)。$ run calc_add 5
exitexit離開 Mini-OS 模擬器 (若在互動式環境)。$ exit
⚙️ 行程管理: run, ps, kill, reset, start, top 📊 排程: sched, aging 🧠 記憶體: mem, load, lsasm, disasm 📨 IPC: ipc_sender, ipc_receiver 🔐 信號量: sem create, sem, mutex_demo 🪲 除錯: deadlock, history, !n, tui

📌 典型操作流程範例

從建立行程到觀察排程、記憶體及 IPC 的完整步驟

$reset
$sched PRIORITY
$aging on
$run count5 5
$run factorial 2
$ps
$start 2
$top
$mem

✅ 說明：重置後設定優先權排程 + 老化，建立兩個行程，start 後將按照優先權執行 (count5 先跑)，top 可觀察切換次數，mem 檢查頁框釋放狀況。

💡 提示：所有「行程名稱」皆可搭配優先權參數（例如 run factorial 4）。SJF 演算法會依據指令條數動態估算執行時間。信號量操作內建於 mutex_demo 中，無需手動呼叫 WAIT/SIGNAL 指令。
