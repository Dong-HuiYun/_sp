⚙️ Mini-OS 測試實驗室

行程 · 排程 · 記憶體 · IPC · 信號量 · 死結偵測 · 組合語言載入器

核心流程 ① 行程生命週期
建立、就緒佇列、執行與上下文切換
run / start / ps 建立 → 排隊 → 執行
$ls
$run hello
$run calc_add 3
$ps
$start
$top

📌 ps 顯示 Ready: [PID:1, PID:2]。start 後 hello 印出 999，calc_add 印出 30。top 看到 ctx_sw 和 wait 皆有數值。

指令擴充 ② 新指令集 · 條件分支與乘除法
count5 CMP + JNZ (帶終止條件)
$reset
$run count5
$start

✅ 印出 1 2 3 4 5，自動終止（不死循環）。

factorial MUL + SUB 階乘
$reset
$run factorial
$start

✅ 印出 120 (5! = 120)。

排程核心 ③ 排程演算法 (Priority / SJF / FIFO)
PRIORITY 高優先權先執行
$reset
$sched PRIORITY
$run hello 1
$run count5 5
$run factorial 3
$start

⚡ 執行順序：count5 (Pri:5) → factorial (Pri:3) → hello (Pri:1)。

SJF 最短作業優先
$reset
$sched SJF
$run factorial
$run hello
$run count5
$start

⏱️ hello(3條) → count5(8條) → factorial(10條)

FIFO 先進先出
$reset
$sched FIFO
$run hello
$run factorial
$run count5
$start

🎯 完全依照 run 的先後順序執行。

防飢餓 ④ Aging — 低優先權動態提升
aging on 避免 starvation
$reset
$sched PRIORITY
$aging on
$run count5 10
$run hello 1
$start 1

📈 hello 初始 Pri:1，等待後出現 `[Aging] PID 2: priority 1 → 2`，top 中 pri 欄位顯示 (+N) 標記。

記憶體管理 ⑤ 分頁分配 / 缺頁中斷
mem 分頁分配與釋放
$reset
$run count5
$run factorial
$mem
$start
$mem

📦 start 前 Used:2，執行完畢後 Used:0（頁框全數釋放）。

page fault Lazy loading 觸發
$reset
$load programs/factorial.asm
$start
$mem

⚠️ 執行到第2頁顯示 `[OS] Page fault: PID 1 vp=1`，mem 顯示 Page faults: 1（若程式超過16條）。

行程通訊 ⑥ IPC 同步訊息傳遞
正常傳送 sender → receiver
$reset
$run ipc_sender
$run ipc_receiver
$start

📨 兩者皆印出 42。

WAITING 狀態 receiver 先等待
$reset
$run ipc_receiver
$run ipc_sender
$start 1
$ps

⏳ ps 顯示 Waiting: [PID:1]，sender 執行後自動喚醒 receiver。

並行控制 ⑦ 信號量 mutex 與阻塞
sem / mutex_demo 互斥鎖保護臨界區
$reset
$sem create 1 mutex
$run mutex_demo
$run mutex_demo
$sem
$start

🔒 第二個行程執行 WAIT 時被 block，sem 顯示 value=-1 waiting=[PID:2]，第一個 SIGNAL 後 PID:2 自動喚醒。

死結偵測 ⑧ 循環等待偵測與解除
deadlock / deadlock resolve 自動偵測循環依賴
$reset
$sem create 1 A
$sem create 1 B
$deadlock
$deadlock resolve

🔍 deadlock 顯示 "No deadlock detected"。若手動建立互相等待則顯示 Cycle: PID:1 → PID:2 → PID:1。

組合語言載入 ⑨ load / lsasm / disasm 反組譯
.asm 載入系統 從 programs/ 載入並反組譯
$reset
$lsasm
$load programs/count5.asm 2
$load programs/factorial.asm 1
$disasm count5
$sched PRIORITY
$start

📂 lsasm 列出 .asm 檔案，load 後根據優先權執行，disasm 顯示反組譯結果。

命令歷史 ⑩ history / !n 重新執行
$run hello
$run count5
$history
$!1
$ps

🕘 history 顯示編號，!1 重新執行 run hello，ps 會多一個 PID。

行程控制 ⑪ kill 終止 Ready/Waiting 行程 + reset
kill ready 終止就緒佇列行程
$reset
$run count5
$run factorial
$kill 2
$ps
$start

✂️ kill 後 ps 只剩 PID:1，只有 count5 執行。

kill waiting 終止阻塞行程
$reset
$run ipc_receiver
$start
$kill 1

💀 顯示 Process 1 killed (was waiting)。

即時儀表板 ⑫ TUI 全螢幕四格監控
$reset
$sem create 1 mutex
$run mutex_demo
$run ipc_sender
$run ipc_receiver
$start 1
$tui

🖥️ 開啟全螢幕儀表板，即時顯示行程狀態、記憶體頁、信號量、IPC 收件匣，每 0.5 秒刷新，按 q 離開。

$ls
