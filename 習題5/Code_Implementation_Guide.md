# 實作範例說明：多執行緒應用

## 1. 銀行存提款模擬 (`bank.cpp`)
這是一個典型的**互斥鎖 (Mutex)** 應用範例。

### 實作重點
- **共享資源**：`BankAccount` 類別中的 `balance_`。
- **機制**：使用 `std::mutex` 保護 `deposit`、`withdraw` 與 `balance` 函式。
- **關鍵語法**：`std::lock_guard<std::mutex> lock(mutex_);`
  - 這是一種 RAII 風格的鎖：建構時自動上鎖，離開函式作用域時自動解鎖，避免忘記解鎖導致的死鎖。

### 執行流程
1. 初始化餘額為 1,000,000。
2. 啟動 2 個存款執行緒（各存 100,000 次）與 2 個提款執行緒（各提 100,000 次）。
3. 由於每個操作都受到 Mutex 保護，最終餘額必能正確回到 1,000,000，不會因競態條件而遺失資料。

## 2. 哲學家用餐問題 (`Dining_philosophers.cpp`)
這是一個示範如何**避免死鎖 (Deadlock Prevention)** 的經典範例。

### 實作重點
- **問題點**：若 5 位哲學家同時拿起左手邊筷子，會導致循環等待，發生死鎖。
- **解決方案（資源排序法）**：
  - 程式計算 `first = min(left, right)` 與 `second = max(left, right)`。
  - 每位哲學家規定**先拿編號小的筷子，再拿編號大的**。
  - 這樣編號最大的 4 號哲學家會與 0 號哲學家競爭「0 號筷子」，打破了循環鏈。

### 執行流程
1. 建立 5 個執行緒代表哲學家，5 個 Mutex 代表筷子。
2. 哲學家進入「思考 -> 拿小號筷子 -> 拿大號筷子 -> 用餐 -> 放下筷子」的循環。
3. 程式能流暢執行直到結束，證明不會發生死鎖。

## 3. 生產者消費者問題 (`Producer_consumer.cpp`)
這是一個示範**執行緒協作 (Thread Synchronization)** 與**條件變數 (Condition Variable)** 的範例。

### 實作重點
- **共享緩衝區**：一個固定容量的 `std::queue`。
- **條件變數**：
  - `not_full_`: 當緩衝區滿了，生產者在此等待。
  - `not_empty_`: 當緩衝區空了，消費者在此等待。
- **關鍵語法**：`wait(lock, predicate)`
  - 執行緒在等待時會釋放 Mutex，直到被通知（`notify_one`）且條件符合時才重新上鎖並繼續。

### 執行流程
1. **生產者**生產資料放入佇列。如果佇列滿了（容量 3），它會暫停執行。
2. **消費者**從佇列取出資料。如果佇列空了，它會暫停執行。
3. 透過 `notify_one()`，生產者放完東西後會叫醒消費者，消費者拿完東西後會叫醒生產者。
4. 最終 10 個物件會被正確生產並由 2 個消費者消耗完畢。
