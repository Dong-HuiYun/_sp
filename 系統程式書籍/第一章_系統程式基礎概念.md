# 第一章：系統程式設計基礎概念

## 1.1 系統程式的定義與範疇

系統程式是指那些為支援和管理電腦硬體與應用程式而設計的軟體。它們直接與硬體互動，提供其他程式運行的環境。主要的系統程式包括：

### 1.1.1 作業系統

作業系統是系統程式的核心，它管理電腦的硬體資源並提供服務給應用程式。常見的作業系統包括：

- Windows：微軟開發的圖形化作業系統
- Linux：開源的類Unix作業系統
- macOS：蘋果公司的作業系統
- Android：基於Linux的移動作業系統
- iOS：蘋果公司的移動作業系統

### 1.1.2 系統工具程式

系統工具程式協助使用者和管理員管理電腦系統，包括：

- 檔案管理工具
- 系統監控工具
- 網路配置工具
- 磁碟管理工具

### 1.1.3 系統服務

系統服務是在背景運行的程式，提供各種系統功能：

- 印表機服務
- 網路服務
- 資料庫服務
- 安全服務

## 1.2 系統層級架構

電腦系統可以分為多個層次，每個層次都有其特定的功能和責任。

### 1.2.1 硬體層

硬體層是系統的最底層，包括：

- **中央處理器（CPU）**：執行指令和處理資料
- **記憶體（RAM）**：臨時儲存資料和程式
- **儲存設備（HDD/SSD）**：永久儲存資料
- **輸入輸出設備**：鍵盤、滑鼠、螢幕、網卡等

### 1.2.2 韌體層

韌體是嵌入在硬體中的軟體，提供硬體和軟體之間的介面：

- **BIOS（基本輸入輸出系統）**：傳統的系統韌體
- **UEFI（統一可延伸韌體介面）**：現代化的韌體標準

### 1.2.3 作業系統層

作業系統是硬體和應用程式之間的橋樑：

```
┌─────────────────────────────────┐
│        應用程式層               │
├─────────────────────────────────┤
│        系統服務層               │
├─────────────────────────────────┤
│        作業系統核心             │
├─────────────────────────────────┤
│        驅動程式層               │
├─────────────────────────────────┤
│        硬體層                   │
└─────────────────────────────────┘
```

### 1.2.4 應用程式層

應用程式層是最上層，包含各種使用者應用軟體：

- 辦公軟體
- 網頁瀏覽器
- 遊戲程式
- 開發工具

## 1.3 系統程式的特點

系統程式設計與應用程式設計有著顯著的不同，主要體現在以下幾個方面：

### 1.3.1 效能優先

系統程式對效能要求極高，因為它們是其他程式運行的基礎。一個低效的系統程式會影響整個系統的效能。

```c
// 高效能的記憶體複製
void *memcpy_optimized(void *dest, const void *src, size_t n) {
    unsigned char *d = (unsigned char *)dest;
    const unsigned char *s = (const unsigned char *)src;
    
    // 使用位元組複製
    for (size_t i = 0; i < n; i++) {
        d[i] = s[i];
    }
    return dest;
}
```

### 1.3.2 直接硬體存取

系統程式通常需要直接與硬體互動，這要求程式設計師深入了解硬體架構。

```c
// 直接記憶體存取範例（需要作業系統許可）
#define PHYSICAL_ADDRESS 0x1000
void *ptr = (void *)PHYSICAL_ADDRESS;
```

### 1.3.3 穩定性要求

系統程式必須高度穩定，一個小小的錯誤可能導致系統崩潰。

### 1.3.4 安全性考量

系統程式處理敏感的系統資源，必須具備完善的安全機制。

## 1.4 系統呼叫介面

系統呼叫是使用者程式請求作業系統服務的標準方式。不同的作業系統提供不同的系統呼叫介面。

### 1.4.1 POSIX 標準

POSIX（可移植作業系統介面）是一個標準化的系統呼叫介面，主要被Unix和Linux系統採用。

```c
#include <unistd.h>
#include <sys/types.h>

// POSIX 系統呼叫範例
int main() {
    pid_t pid = fork();  // 創建新行程
    
    if (pid == 0) {
        // 子行程
        execlp("ls", "ls", "-la", NULL);
    } else if (pid > 0) {
        // 父行程
        wait(NULL);
    }
    
    return 0;
}
```

### 1.4.2 Windows API

Windows作業系統使用Win32 API作為其系統呼叫介面。

```c
#include <windows.h>

// Windows API 範例
int main() {
    STARTUPINFOA si;
    PROCESS_INFORMATION pi;
    
    ZeroMemory(&si, sizeof(si));
    si.cb = sizeof(si);
    ZeroMemory(&pi, sizeof(pi));
    
    // 創建新行程
    if (!CreateProcessA(
        "C:\\Windows\\System32\\cmd.exe",
        NULL,
        NULL,
        NULL,
        FALSE,
        0,
        NULL,
        NULL,
        &si,
        &pi
    )) {
        return 1;
    }
    
    WaitForSingleObject(pi.hProcess, INFINITE);
    CloseHandle(pi.hProcess);
    CloseHandle(pi.hThread);
    
    return 0;
}
```

## 1.5 常見的系統程式類型

### 1.5.1 編譯器

編譯器將高階語言程式翻譯成機器碼或中間碼。

```c
// 簡單的表達式解析器概念
typedef enum {
    TOKEN_NUMBER,
    TOKEN_PLUS,
    TOKEN_MINUS,
    TOKEN_MULTIPLY,
    TOKEN_DIVIDE,
    TOKEN_LPAREN,
    TOKEN_RPAREN,
    TOKEN_EOF
} TokenType;

typedef struct {
    TokenType type;
    double value;
} Token;
```

### 1.5.2 連結器

連結器將多個目標檔案合併成一個可執行檔。

### 1.5.3 載入器

載入器將可執行檔載入記憶體並準備執行。

```c
// 簡化的載入器概念
typedef struct {
    unsigned int entry_point;    // 入口點位址
    unsigned int code_size;      // 程式碼大小
    unsigned int data_size;      // 資料大小
    unsigned int bss_size;       // BSS區段大小
} ExecutableHeader;

void load_program(const char *filename) {
    ExecutableHeader header;
    FILE *fp = fopen(filename, "rb");
    
    fread(&header, sizeof(header), 1, fp);
    
    void *code = malloc(header.code_size);
    void *data = malloc(header.data_size);
    void *bss = malloc(header.bss_size);
    
    fread(code, header.code_size, 1, fp);
    fread(data, header.data_size, 1, fp);
    
    fclose(fp);
}
```

### 1.5.4 驅動程式

驅動程式是作業系統和硬體之間的橋樑。

```c
// Linux 核心模組範例結構
#include <linux/module.h>
#include <linux/kernel.h>

int init_module(void) {
    printk(KERN_INFO "Hello World - Driver loaded\n");
    return 0;
}

void cleanup_module(void) {
    printk(KERN_INFO "Goodbye World - Driver unloaded\n");
}

MODULE_LICENSE("GPL");
MODULE_AUTHOR("Author");
MODULE_DESCRIPTION("Simple Driver");
```

### 1.5.5 網路協議棧

網路協議棧實現網路通訊的各層協議。

## 1.6 開發環境設置

### 1.6.1 Linux 開發環境

在Linux系統上進行系統程式設計需要以下工具：

```bash
# 安裝基本開發工具
sudo apt-get update
sudo apt-get install build-essential

# 安裝除錯工具
sudo apt-get install gdb

# 安裝系統程式設計相關庫
sudo apt-get install manpages-dev
```

### 1.6.2 Windows 開發環境

在Windows系統上可以使用以下工具：

- Visual Studio：完整的整合開發環境
- MinGW：GNU工具鏈的Windows移植
- WSL（Windows Subsystem for Linux）：在Windows中運行Linux

### 1.6.3 交叉編譯環境

對於嵌入式系統開發，需要設置交叉編譯環境：

```bash
# 安裝ARM交叉編譯工具鏈
sudo apt-get install gcc-arm-none-eabi

# 編譯ARM程式
arm-none-eabi-gcc -o firmware.elf firmware.c
```

## 1.7 系統程式設計的挑戰

### 1.7.1 可移植性

不同的硬體架構和作業系統有不同的特性，編寫可移植的系統程式需要仔細考慮這些差異。

### 1.7.2 併發處理

現代系統需要處理大量的併發操作，這帶來了同步和競爭條件的挑戰。

### 1.7.3 記憶體管理

正確管理記憶體，避免洩漏和碎片，是系統程式設計的重要課題。

### 1.7.4 錯誤處理

系統程式必須優雅地處理各種錯誤情況，避免系統崩潰。
