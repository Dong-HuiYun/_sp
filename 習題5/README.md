# 多執行緒程式設計核心概念

本文件介紹並行計算（Concurrent Computing）中的四大核心概念：執行緒、競態條件、互斥鎖與死鎖。

## 1. 執行緒 (Thread)
執行緒是作業系統能夠進行運算排程的最小單位。
- **特性**：一個行程（Process）可以包含多個執行緒，它們共享該行程的記憶體空間（如全域變數、堆積記憶體）。
- **目的**：利用多核處理器的優勢，同時處理多個任務，提高程式執行效率或反應速度。

## 2. 競態條件 (Race Condition)
當多個執行緒同時存取並修改同一個共享資源，且最終結果取決於執行緒執行的「順序」或「時機」時，就會發生競態條件。
- **後果**：導致資料不一致、計算結果錯誤（例如：明明存了兩次 100 元，餘額卻只增加 100 元）。
- **關鍵區塊 (Critical Section)**：指程式中存取共享資源的那段程式碼，必須確保同一時間只有一個執行緒能進入。

## 3. 互斥鎖 (Mutex)
互斥鎖（Mutual Exclusion）是解決競態條件最常用的同步機制。
- **原理**：像是一把鑰匙。執行緒在進入關鍵區塊前必須先「上鎖」(Lock)，執行完後「解鎖」(Unlock)。
- **效果**：如果鎖已被佔用，其他執行緒必須等待（阻塞），直到持有鎖的執行緒釋放它。
- **C++ 工具**：`std::mutex` 以及自動管理生命週期的 `std::lock_guard`。

## 4. 死鎖 (Deadlock)
當兩個或多個執行緒互相等待對方釋放資源，導致所有相關執行緒都無法繼續執行時，稱為死鎖。
- **經典範例**：
  - 執行緒 A 拿著鎖 1，等待鎖 2。
  - 執行緒 B 拿著鎖 2，等待鎖 1。
- **解決方案**：
  - **資源排序法**：規定所有執行緒必須按相同的順序請求鎖。
  - **超時機制**：嘗試拿鎖，失敗則放掉已有的鎖。
  - **使用更高階的同步工具**：如 `std::scoped_lock`（C++17）。
