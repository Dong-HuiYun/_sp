# 哲學家用餐模擬 — Dining Philosophers

示範死鎖（Deadlock）如何在多執行緒環境中自然發生，以及如何用「資源排序法」在不降低並行度的前提下徹底預防死鎖。

---

## 問題背景

5 位哲學家圍坐圓桌，每兩人之間放一支筷子，共 5 支。哲學家只做兩件事：思考和吃飯。吃飯前必須同時拿到左右兩支筷子，吃完放回去，再繼續思考。

```
        Aristotle (0)
      筷子0        筷子4
  Plato (1)        Kant (4)
      筷子1        筷子3
    Socrates (2)  Descartes (3)
           筷子2
```

每支筷子對應一個 mutex，拿起筷子就是鎖住 mutex，放下就是解鎖。

---

## 問題所在：死鎖如何發生

若每個人都先拿左手筷子，再去等右手筷子，會形成以下的循環等待：

```
Aristotle  拿著筷子 0，等待筷子 1
Plato      拿著筷子 1，等待筷子 2
Socrates   拿著筷子 2，等待筷子 3
Descartes  拿著筷子 3，等待筷子 4
Kant       拿著筷子 4，等待筷子 0  ← 形成循環
```

每個人都拿著一支、等著另一支，但另一支永遠不會被釋放，因為拿著它的人也在等別人。程式永久卡住，這就是死鎖。

死鎖的四個必要條件在此全部成立：

| 條件 | 本題對應情況 |
|------|-------------|
| 互斥 | 每支筷子同時只能一人持有 |
| 持有並等待 | 拿著一支，等待另一支 |
| 不可搶占 | 筷子不能被強行奪走 |
| 循環等待 | 五人形成等待的環 |

只要打破其中一個條件，死鎖就不會發生。

---

## 解決方案：資源排序法

為所有筷子固定編號 0 到 4，規定一條全域規則：

> **每個人都必須先拿編號小的筷子，再拿編號大的筷子。**

套用這條規則後，每位哲學家的取筷順序如下：

| 哲學家 | 左手筷子 | 右手筷子 | 實際取筷順序 |
|--------|----------|----------|-------------|
| Aristotle (0) | 筷子 0 | 筷子 1 | 先 0，再 1 |
| Plato (1) | 筷子 1 | 筷子 2 | 先 1，再 2 |
| Socrates (2) | 筷子 2 | 筷子 3 | 先 2，再 3 |
| Descartes (3) | 筷子 3 | 筷子 4 | 先 3，再 4 |
| **Kant (4)** | **筷子 4** | **筷子 0** | **先 0，再 4（順序對調！）** |

前四位哲學家的順序本來就是小到大，不受影響。只有 Kant 的順序被改變：他原本會先拿自己左手的筷子 4，現在改為先去搶筷子 0。

這個改變打破了循環等待。筷子 0 同時只有一個人能拿到，不管是 Aristotle 還是 Kant 先搶到，另一個人手裡什麼都沒有，不佔用任何資源，不阻擋其他人，等待的環就此斷開。

## 程式設計

### 全域資源

| 變數 | 說明 |
|------|------|
| `chopsticks[5]` | 5 個 mutex，各代表一支筷子 |
| `total_meals` | 原子計數器，記錄總用餐次數，供最後驗證 |

### 哲學家的行為（每輪）

```
1. 思考：睡眠一段時間（各人不同，避免同時醒來搶筷子）
2. 計算自己應先拿哪支：first = min(左, 右)，second = max(左, 右)
3. 鎖住 chopsticks[first]
4. 鎖住 chopsticks[second]
5. 用餐：睡眠 80ms，total_meals 加一
6. 離開作用域，兩個 lock_guard 自動解鎖（放下筷子）
```

重複以上流程 3 次後，這位哲學家的執行緒結束。

### 為何不需要手動放筷子

程式使用 `std::lock_guard`，它的析構函式會自動解鎖。當每一輪迴圈結束，兩個 lock_guard 的生命週期結束，兩支筷子自動被放回，不需要手動呼叫任何解鎖函式。

## 編譯與執行

進入`wsl`環境

```bash

g++ dining_philosophers.cpp -o dining_philosophers
./dining_philosophers

```

## 預期輸出

```
=== 哲學家用餐問題（資源排序法，避免死鎖）===
哲學家數量 = 5，每人用餐 3 次

[Aristotle] 思考中...
[Plato    ] 思考中...
...
[Aristotle] 拿起筷子 #0
[Aristotle] 拿起筷子 #1 → 開始用餐 (第 1 次)
...
[Done] 所有哲學家用餐完畢，總計 15 次，無死鎖發生。
```

程式正常結束（不卡住）且總計 15 次（5 人 × 3 輪），代表無死鎖發生。

## 與其他解法的比較

| 解法 | 原理 | 優點 | 缺點 |
|------|------|------|------|
| **資源排序法（本程式）** | 固定取得順序，破除循環等待 | 實作簡單，效能佳 | 需預先知道所有資源 |
| 仲裁者法 | 設一個管理者統一分配筷子 | 靈活 | 管理者成為效能瓶頸 |
| 一次性取得所有資源 | 兩支都能拿才拿，否則都不拿 | 簡單可靠 | 降低並行度 |
| Chandy/Misra 演算法 | 以乾淨/骯髒標記傳遞筷子 | 完全分散式，公平 | 實作複雜 |
