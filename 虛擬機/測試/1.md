把所有檔案放到同一個目錄，然後 `python3 main.py` 啟動。以下是每個功能的完整測試流程：

---

## 1. 基本功能

```
ls
```
確認看到 7 個程式（calc_add、counter、hello、count5、factorial、ipc_sender、ipc_receiver）

```
run hello
run calc_add 3
ps
```
預期 `ps` 輸出：
```
Running : None
Ready   : ['PID:1(Pri:1)', 'PID:2(Pri:3)']
Waiting : (empty)
```

```
start
```
啟動排程器，應該看到兩個 process 依序執行並輸出結果，hello 印出 `999`，calc_add 印出 `30`。

---

## 2. 新指令集（CMP / JNZ / MUL / SUB）

```
reset
run count5
start
```
預期輸出從 1 數到 5 然後自動停止（不會像 loop 一樣死迴圈）：
```
[VM Output] R1: 1
[VM Output] R1: 2
[VM Output] R1: 3
[VM Output] R1: 4
[VM Output] R1: 5
[OS] Process 1 finished.
```

```
reset
run factorial
start
```
預期輸出 `[VM Output] R1: 120`（5! = 120）

---

## 3. 排程演算法切換

**測試 PRIORITY**（高優先權的先跑）：
```
reset
sched PRIORITY
run calc_add 1
run hello 5
run counter 3
ps
start
```
預期順序：hello（Pri:5）→ counter（Pri:3）→ calc_add（Pri:1）

**測試 SJF**（指令數少的先跑）：
```
reset
sched SJF
run factorial
run hello
run count5
start
```
預期順序：hello（3條）→ count5（8條）→ factorial（10條）

**測試 FIFO**：
```
reset
sched FIFO
run hello
run calc_add
run counter
start
```
預期按照 run 的先後順序執行。

---

## 4. Process 狀態機 + 統計（top）

```
reset
run count5 2
run factorial 1
start 1
```
等幾秒後（time quantum=1，切換頻繁），輸入：
```
top
```
預期看到每個 process 的詳細資訊，context switch 次數 > 0，wait 時間 > 0：
```
PID   STATE        PRI   INSTR    CTX_SW   WAIT       AGE
-----------------------------------------------------------------
PID:1  state:RUNNING    pri:2   instr:4   ctx_sw:2   wait:1.20s  age:5.3s
PID:2  state:READY      pri:1   instr:3   ctx_sw:1   wait:2.10s  age:5.3s
```

## 5. 虛擬記憶體分頁（mem）

```
reset
run count5
run factorial
run hello
mem
```
預期看到 3 個 process 各占一個頁框：
```
[MemMgr] Total frames: 64 | Used: 3 | Free: 61
  PID 1: pages=[0] → frames=[0]
  PID 2: pages=[0] → frames=[1]
  PID 3: pages=[0] → frames=[2]
```

然後啟動並等待執行完畢：
```
start
mem
```
預期執行完後 `Used: 0 | Free: 64`，頁框全部釋放。

## 6. IPC（行程間通訊）

這個需要 **sender 先建立（PID 1），receiver 後建立（PID 2）**，因為 sender 的 SEND 指令寫死目標是 PID 2：
```
reset
run ipc_sender
run ipc_receiver
ipc
start
```

`ipc` 指令先確認收件匣是空的。啟動後預期流程：
1. PID 1（sender）執行，把 `42` 送進 PID 2 的收件匣
2. PID 2（receiver）執行 RECV，成功取得值，印出 `42`

預期輸出：
```
[IPC] PID 1 → PID 2: value=42
[VM Output] R1: 42          ← sender 印出已送出的值
[OS] Process 1 finished.

[VM] PID 2 RECV: R1=42 (from PID 1)
[VM Output] R1: 42          ← receiver 印出收到的值
[OS] Process 2 finished.
```

**測試 WAITING 狀態**（receiver 先跑，收件匣是空的）：
```
reset
run ipc_receiver
run ipc_sender
ps
start 1
```
啟動後輸入 `ps`，預期看到 receiver 進入 WAITING：
```
Running : None
Ready   : ['PID:2(Pri:1)']
Waiting : ['PID:1']
```
接著 sender 執行完 SEND，OS 自動把 receiver 喚醒回 READY 再執行。

---

## 7. Kill 與 Reset

**kill 執行中的 process**（需要在 start 之後快速輸入）：
```
reset
run count5
run factorial
start 1
kill 1
ps
```
預期 PID 1 消失，PID 2 繼續跑。

**kill WAITING 中的 process**：
```
reset
run ipc_receiver
start
kill 1
```
預期看到 `[OS] Process 1 killed (was waiting).`

**reset**：
```
reset
ps
mem
top
```
三個都應該顯示空，`next_pid` 重置（下次 run 會從 PID 1 開始）。
