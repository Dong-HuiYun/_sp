# 第二章：程式語言與編譯器

## 2.1 程式語言的層次

程式語言可以根據其抽象層次分為多個等級，從最接近機器語言的低階語言到最接近人類語言的高階語言。

### 2.1.1 機器語言

機器語言是電腦CPU能夠直接執行的二進制指令集，每條指令都是一串0和1。

```
範例（x86架構）：
10111000 00000001 00000000  # mov eax, 1
00000101 00000000 00000000  # add eax, 0
11000011                      # ret
```

### 2.1.2 組合語言

組合語言是機器語言的符號化表示，使用助記符號代替二進制指令。

```asm
; x86組合語言範例
section .text
global _start

_start:
    mov eax, 1      ; 系統呼叫編號
    mov ebx, 0      ; 退出碼
    int 0x80        ; 觸發系統呼叫
```

### 2.1.3 高階語言

高階語言提供更抽象的語法，使程式設計更為簡便：

- **C語言**：系統程式設計的主流語言
- **C++**：支援物件導向的系統程式語言
- **Rust**：現代系統程式語言，注重記憶體安全
- **Go**：Google開發的併發程式語言

```c
// C語言範例
#include <stdio.h>

int main() {
    printf("Hello, System Programming!\n");
    return 0;
}
```

## 2.2 編譯過程概述

編譯是將高階語言程式翻譯成機器語言的過程。這個過程分為多個階段，每個階段都有其特定的功能。

```
原始程式碼
    ↓
┌─────────────────┐
│  詞法分析        │
└─────────────────┘
    ↓
┌─────────────────┐
│  語法分析        │
└─────────────────┘
    ↓
┌─────────────────┐
│  語意分析        │
└─────────────────┘
    ↓
┌─────────────────┐
│  中間碼生成      │
└─────────────────┘
    ↓
┌─────────────────┐
│  優化            │
└─────────────────┘
    ↓
┌─────────────────┐
│  目標碼生成      │
└─────────────────┘
    ↓
目標檔案（機器碼）
```

## 2.3 詞法分析

詞法分析是編譯過程的第一階段，其任務是將原始程式碼字元流轉換成詞素（Token）序列。

### 2.3.1 詞素的類型

常見的詞素類型包括：

```c
typedef enum {
    TOKEN_IDENTIFIER,     // 識別符
    TOKEN_KEYWORD,        // 關鍵字
    TOKEN_INTEGER,        // 整數常數
    TOKEN_FLOAT,          // 浮點常數
    TOKEN_STRING,         // 字串常數
    TOKEN_OPERATOR,       // 運算子
    TOKEN_PUNCTUATOR,     // 標點符號
    TOKEN_COMMENT,        // 註解
    TOKEN_WHITESPACE,     // 空白字元
    TOKEN_NEWLINE,        // 換行符
    TOKEN_EOF            // 檔案結束
} TokenType;
```

### 2.3.2 詞法分析器實現

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <ctype.h>

typedef struct {
    TokenType type;
    char lexeme[100];
    int value;
    int line;
    int column;
} Token;

typedef struct {
    const char *source;
    size_t position;
    int line;
    int column;
} Lexer;

void lexer_init(Lexer *lexer, const char *source) {
    lexer->source = source;
    lexer->position = 0;
    lexer->line = 1;
    lexer->column = 1;
}

char current_char(Lexer *lexer) {
    return lexer->source[lexer->position];
}

char advance(Lexer *lexer) {
    char c = lexer->source[lexer->position++];
    if (c == '\n') {
        lexer->line++;
        lexer->column = 1;
    } else {
        lexer->column++;
    }
    return c;
}

Token next_token(Lexer *lexer) {
    Token token = {TOKEN_EOF, "", 0, lexer->line, lexer->column};
    
    while (isspace(current_char(lexer))) {
        advance(lexer);
    }
    
    if (isalpha(current_char(lexer)) || current_char(lexer) == '_') {
        int i = 0;
        while (isalnum(current_char(lexer)) || current_char(lexer) == '_') {
            token.lexeme[i++] = advance(lexer);
        }
        token.lexeme[i] = '\0';
        token.type = TOKEN_IDENTIFIER;
    } else if (isdigit(current_char(lexer))) {
        int i = 0;
        while (isdigit(current_char(lexer))) {
            token.lexeme[i++] = advance(lexer);
        }
        token.lexeme[i] = '\0';
        token.type = TOKEN_INTEGER;
        token.value = atoi(token.lexeme);
    } else if (current_char(lexer) == '+') {
        token.type = TOKEN_OPERATOR;
        token.lexeme[0] = advance(lexer);
        token.lexeme[1] = '\0';
    }
    
    return token;
}
```

### 2.3.3 正規表達式在詞法分析中的應用

正規表達式常用於定義詞素的模式：

```
IDENTIFIER  = [a-zA-Z_][a-zA-Z0-9_]*
INTEGER     = [0-9]+
FLOAT       = [0-9]+\.[0-9]+
STRING      = \"[^\"]*\"
COMMENT     = \/\/.*|\/\*[\s\S]*?\*\/
```

## 2.4 語法分析

語法分析將詞素序列轉換成語法樹，檢查程式的語法結構是否正確。

### 2.4.1 上下文無關文法

文法是描述語言語法規則的形式化系統。

```bnf
<程式>      ::= <宣告列表>
<宣告列表>  ::= <宣告> | <宣告列表> <宣告>
<宣告>      ::= <型別> <識別符> ';' 
            | <型別> <識別符> '=' <運算式> ';'

<運算式>    ::= <項> | <運算式> '+' <項> | <運算式> '-' <項>
<項>        ::= <因子> | <項> '*' <因子> | <項> '/' <因子>
<因子>      ::= <識別符> | <整數> | '(' <運算式> ')'

<型別>      ::= 'int' | 'float' | 'char'
```

### 2.4.2 語法樹節點

### 2.4.3 遞歸下降剖析器

```c
typedef struct {
    Token *tokens;
    int position;
    int size;
} Parser;

Token *current_token(Parser *parser) {
    if (parser->position < parser->size) {
        return &parser->tokens[parser->position];
    }
    return NULL;
}

Token *eat(Parser *parser, TokenType type) {
    Token *token = current_token(parser);
    if (token && token->type == type) {
        parser->position++;
        return token;
    }
    return NULL;
}

ASTNode* parse_expression(Parser *parser) {
    return parse_addition(parser);
}

ASTNode* parse_addition(Parser *parser) {
    ASTNode *node = parse_multiplication(parser);
    
    while (current_token(parser) && 
           (current_token(parser)->type == TOKEN_PLUS ||
            current_token(parser)->type == TOKEN_MINUS)) {
        Token *op = eat(parser, current_token(parser)->type);
        ASTNode *right = parse_multiplication(parser);
        
        ASTNode *parent = create_node(NODE_BINARY_OP);
        parent->left = node;
        parent->right = right;
        strcpy(parent->name, op->lexeme);
        node = parent;
    }
    
    return node;
}

ASTNode* parse_multiplication(Parser *parser) {
    ASTNode *node = parse_primary(parser);
    
    while (current_token(parser) && 
           (current_token(parser)->type == TOKEN_MULTIPLY ||
            current_token(parser)->type == TOKEN_DIVIDE)) {
        Token *op = eat(parser, current_token(parser)->type);
        ASTNode *right = parse_primary(parser);
        
        ASTNode *parent = create_node(NODE_BINARY_OP);
        parent->left = node;
        parent->right = right;
        strcpy(parent->name, op->lexeme);
        node = parent;
    }
    
    return node;
}

ASTNode* parse_primary(Parser *parser) {
    Token *token = current_token(parser);
    
    if (token->type == TOKEN_INTEGER) {
        eat(parser, TOKEN_INTEGER);
        ASTNode *node = create_node(NODE_INTEGER);
        node->int_value = token->value;
        return node;
    }
    
    if (token->type == TOKEN_LPAREN) {
        eat(parser, TOKEN_LPAREN);
        ASTNode *node = parse_expression(parser);
        eat(parser, TOKEN_RPAREN);
        return node;
    }
    
    return NULL;
}
```

## 2.5 語意分析

語意分析檢查程式的語意正確性，包括型別檢查、作用域管理等。

### 2.5.1 符號表

符號表儲存程式中所有識別符的資訊：

```c
typedef enum {
    SYMBOL_VARIABLE,
    SYMBOL_FUNCTION,
    SYMBOL_TYPE,
    SYMBOL_CONSTANT
} SymbolType;

typedef struct Symbol {
    char name[100];
    SymbolType type;
    int data_type;
    int scope_level;
    void *info;
    struct Symbol *next;
} Symbol;

typedef struct {
    Symbol *symbols[1000];
    int count;
} SymbolTable;

void symbol_table_init(SymbolTable *table) {
    table->count = 0;
    memset(table->symbols, 0, sizeof(table->symbols));
}

unsigned int hash(const char *name) {
    unsigned int hash = 5381;
    int c;
    while ((c = *name++)) {
        hash = ((hash << 5) + hash) + c;
    }
    return hash % 1000;
}

Symbol* symbol_lookup(SymbolTable *table, const char *name) {
    unsigned int index = hash(name);
    Symbol *symbol = table->symbols[index];
    
    while (symbol) {
        if (strcmp(symbol->name, name) == 0) {
            return symbol;
        }
        symbol = symbol->next;
    }
    return NULL;
}

void symbol_insert(SymbolTable *table, const char *name, SymbolType sym_type, int data_type) {
    unsigned int index = hash(name);
    Symbol *symbol = (Symbol*)malloc(sizeof(Symbol));
    
    strcpy(symbol->name, name);
    symbol->type = sym_type;
    symbol->data_type = data_type;
    symbol->scope_level = 0;
    symbol->next = table->symbols[index];
    
    table->symbols[index] = symbol;
    table->count++;
}
```

## 2.6 中間碼生成

中間碼是介於原始碼和目標碼之間的一種表示形式，便於優化和跨平台移植。

### 2.6.1 三地址碼

三地址碼是最常見的中間表示形式：

```c
typedef enum {
    TAC_ADD,
    TAC_SUB,
    TAC_MUL,
    TAC_DIV,
    TAC_ASSIGN,
    TAC_LOAD,
    TAC_STORE,
    TAC_LABEL,
    TAC_JUMP,
    TAC_IF_EQ,
    TAC_IF_NE,
    TAC_IF_LT,
    TAC_IF_GT,
    TAC_CALL,
    TAC_RETURN
} TACOp;

typedef struct TACInstruction {
    TACOp op;
    char result[50];
    char arg1[50];
    char arg2[50];
    char label[50];
    struct TACInstruction *next;
} TACInstruction;

TACInstruction* create_tac(TACOp op, char *result, char *arg1, char *arg2) {
    TACInstruction *inst = (TACInstruction*)malloc(sizeof(TACInstruction));
    inst->op = op;
    if (result) strcpy(inst->result, result);
    if (arg1) strcpy(inst->arg1, arg1);
    if (arg2) strcpy(inst->arg2, arg2);
    inst->next = NULL;
    return inst;
}

void emit_tac(TACInstruction **head, TACInstruction **tail, TACInstruction *inst) {
    if (*tail) {
        (*tail)->next = inst;
    } else {
        *head = inst;
    }
    *tail = inst;
}
```

### 2.6.2 語法樹到中間碼的轉換

```c
void generate_tac(ASTNode *node, TACInstruction **code) {
    if (!node) return;

            sprintf(result_temp, "t%d", temp_counter);

            if (strcmp(node->name, "+") == 0) {
                printf("%s = %s + %s\n", result_temp, left_temp, right_temp);
            } else if (strcmp(node->name, "-") == 0) {
                printf("%s = %s - %s\n", result_temp, left_temp, right_temp);
            } else if (strcmp(node->name, "*") == 0) {
                printf("%s = %s * %s\n", result_temp, left_temp, right_temp);
            } else if (strcmp(node->name, "/") == 0) {
                printf("%s = %s / %s\n", result_temp, left_temp, right_temp);
            }
            break;
    }
}
```

## 2.7 程式碼優化

程式碼優化旨在提高程式的執行效率、減少資源消耗。

### 2.7.1 常數摺疊

```c
TACInstruction* constant_folding(TACInstruction *code) {
    TACInstruction *result = NULL;
    TACInstruction *tail = NULL;
    
    while (code) {
        if (code->op == TAC_ADD || code->op == TAC_SUB ||
            code->op == TAC_MUL || code->op == TAC_DIV) {
            
            int val1 = atoi(code->arg1);
            int val2 = atoi(code->arg2);
            
            if (isdigit(code->arg1[0]) && isdigit(code->arg2[0])) {
                int result_val;
                switch (code->op) {
                    case TAC_ADD: result_val = val1 + val2; break;
                    case TAC_SUB: result_val = val1 - val2; break;
                    case TAC_MUL: result_val = val1 * val2; break;
                    case TAC_DIV: result_val = val1 / val2; break;
                }
                sprintf(code->result, "%d", result_val);
                code->op = TAC_ASSIGN;
                code->arg1[0] = '\0';
                code->arg2[0] = '\0';
            }
        }
        
        if (!result) {
            result = code;
            tail = code;
        } else {
            tail->next = code;
            tail = code;
        }
        code = code->next;
    }
    
    return result;
}
```

### 2.7.2 死碼消除

```c
int is_used_later(TACInstruction *code, const char *var) {
    while (code) {
        if (code->arg1 && strcmp(code->arg1, var) == 0) return 1;
        if (code->arg2 && strcmp(code->arg2, var) == 0) return 1;
        code = code->next;
    }
    return 0;
}

TACInstruction* dead_code_elimination(TACInstruction *code) {
    TACInstruction *result = NULL;
    TACInstruction *tail = NULL;
    
    while (code) {
        if (code->op == TAC_ASSIGN) {
            if (!is_used_later(code->next, code->result)) {
                code = code->next;
                continue;
            }
        }
        
        if (!result) {
            result = code;
            tail = code;
        } else {
            tail->next = code;
            tail = code;
        }
        code = code->next;
    }
    
    if (tail) tail->next = NULL;
    return result;
}
```

### 2.7.3 循環不變量移動

```c
TACInstruction* loop_invariant_motion(TACInstruction *code, TACInstruction *loop_start) {
    TACInstruction *invariants = NULL;
    TACInstruction *invariant_tail = NULL;
    TACInstruction *remaining = NULL;
    TACInstruction *remaining_tail = NULL;
    TACInstruction *in_loop = loop_start;
    
    while (in_loop) {
        if (is_loop_invariant(in_loop)) {
            if (!invariants) {
                invariants = in_loop;
                invariant_tail = in_loop;
            } else {
                invariant_tail->next = in_loop;
                invariant_tail = in_loop;
            }
        } else {
            if (!remaining) {
                remaining = in_loop;
                remaining_tail = in_loop;
            } else {
                remaining_tail->next = in_loop;
                remaining_tail = in_loop;
            }
        }
        in_loop = in_loop->next;
    }
    
    if (invariant_tail) invariant_tail->next = remaining;
    if (remaining_tail) remaining_tail->next = NULL;
    
    return invariants ? invariants : remaining;
}
```

## 2.8 目標碼生成

目標碼生成是編譯的最後階段，將中間碼轉換成特定架構的機器碼。

```c
Register registers[] = {
```

```c
    int defined;

int resolve_symbols(LinkerSymbol *symbols, int count) {
    for (int i = 0; i < count; i++) {
        if (!symbols[i].defined) {
            for (int j = 0; j < count; j++) {
                if (i != j && 
                    strcmp(symbols[i].name, symbols[j].name) == 0 &&
                    symbols[j].defined) {
                    symbols[i].address = symbols[j].address;
                    symbols[i].defined = 1;
                    break;

            if (!symbols[i].defined) {
                fprintf(stderr, "Undefined symbol: %s\n", symbols[i].name);
                return -1;
            }
        }
    }

}
```
