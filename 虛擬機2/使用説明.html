Mini-OS test guide — commands organized by feature

1 — Basic process lifecycle
run / start / ps Create, queue, execute
$ls
$run hello
$run calc_add 3
$ps
$start
$top

ps 顯示 Ready: [PID:1, PID:2]。start 後 hello 印出 999，calc_add 印出 30。top 看到 ctx_sw 和 wait 都有數值。

2 — New instruction set
count5 CMP + JNZ (有終止條件)
$reset
$run count5
$start

印出 1 2 3 4 5，自動停止（不死循環）。

factorial MUL + SUB
$reset
$run factorial
$start

印出 120（5! = 120）。

3 — Scheduler algorithms
PRIORITY 高優先權先執行
$reset
$sched PRIORITY
$run hello 1
$run count5 5
$run factorial 3
$start

順序：count5（Pri:5）→ factorial（Pri:3）→ hello（Pri:1）。

SJF 最短工作優先
$reset
$sched SJF
$run factorial
$run hello
$run count5
$start

hello(3條) → count5(8條) → factorial(10條)。

FIFO 先進先出
$reset
$sched FIFO
$run hello
$run factorial
$run count5
$start

完全按照 run 的先後順序執行。

4 — Aging（防 starvation）
aging 低優先權自動升級
$reset
$sched PRIORITY
$aging on
$run count5 10
$run hello 1
$start 1

hello 一開始 Pri:1 被排在後面，等一段時間後看到 [Aging] PID 2: priority 1 → 2，top 顯示 pri 欄位有 (+N) 標記。

5 — Memory & page fault
mem 分頁分配與釋放
$reset
$run count5
$run factorial
$mem
$start
$mem

start 前 Used:2，執行完後 Used:0（頁框全部釋放）。

page fault Lazy loading 觸發
$reset
$load programs/factorial.asm
$start
$mem

執行到第 2 頁時出現 [OS] Page fault: PID 1 vp=1，mem 顯示 Page faults: 1（若程式超過 16 條）。

6 — IPC（行程間通訊）
正常傳送 sender → receiver
$reset
$run ipc_sender
$run ipc_receiver
$start

兩個都印出 42。

WAITING 狀態 receiver 先等
$reset
$run ipc_receiver
$run ipc_sender
$start 1
$ps

ps 看到 Waiting: [PID:1]，sender 執行後自動喚醒 receiver。

7 — Semaphore（互斥鎖）
sem mutex 保護 critical section
$reset
$sem create 1 mutex
$run mutex_demo
$run mutex_demo
$sem
$start

第二個 process 執行 WAIT 時被 block，sem 顯示 value=-1 waiting=[PID:2]，第一個 SIGNAL 後 PID:2 自動喚醒。

8 — Deadlock 偵測
deadlock 手動偵測與自動解除
$reset
$sem create 1 A
$sem create 1 B
$deadlock
$deadlock resolve

deadlock 顯示 No deadlock detected（目前無循環）。若手動把 process 放入互相等待狀態再偵測，會顯示 Cycle: PID:1 → PID:2 → PID:1。

9 — Assembler（.asm 載入器）
load / lsasm / disasm
$reset
$lsasm
$load programs/count5.asm 2
$load programs/factorial.asm 1
$disasm count5
$sched PRIORITY
$start

lsasm 列出 programs/ 目錄的 .asm 檔，load 後 count5 因 Pri:2 先執行，印 1–5，再執行 factorial 印 120。disasm 顯示反組譯結果。

10 — History（指令歷史）
history / !n 查看與重執行
$run hello
$run count5
$history
$!1
$ps

history 列出編號。!1 重新執行 run hello，ps 會多一個 PID。

11 — kill / reset
kill 終止 READY 中的行程
$reset
$run count5
$run factorial
$kill 2
$ps
$start

kill 後 ps 只剩 PID:1，只有 count5 執行。

kill 終止 WAITING 行程
$reset
$run ipc_receiver
$start
$kill 1

顯示 Process 1 killed (was waiting)。

12 — TUI 即時儀表板
tui 四格即時畫面
$reset
$sem create 1 mutex
$run mutex_demo
$run ipc_sender
$run ipc_receiver
$start 1
$tui

開啟全螢幕儀表板，同時看到 process 狀態、記憶體分頁、semaphore value、IPC 收件匣，每 0.5 秒刷新。按 q 離開。

一鍵全功能測試（複製貼入）
$ls
$sched PRIORITY
$aging on
$sem create 1 mutex
$run count5 3
$run factorial 1
$run ipc_sender 2
$run ipc_receiver 2
$run mutex_demo 2
$ps
$mem
$sem
$start 2
$top
$mem
$history
$lsasm
$load programs/count5.asm
$disasm factorial
$deadlock
$reset
$exit
